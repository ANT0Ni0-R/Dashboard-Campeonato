# New Mart Model

Create a new mart model for BU and entity specified in `$ARGUMENTS`.

## Conventions

1. **File path**: `models/bigquery/mart/[bu]/mrt_[bu]__[entity].sql`
2. **File name**: `mrt_[bu]__[entity].sql` (plural entity name)
3. **Schema**: Generated by macro as `mart_[bu]`

## Template

```sql
{{
    config(
        materialized = "table",
        meta = {'owner': '<user>'},
        tags = ["<lowest-frequency-source-tag>"]
    )
}}

-- Overview: <brief description of first source>
WITH source_a AS (
    SELECT *
    FROM {{ ref('stg_source__table_a') }}
)

-- Overview: <brief description of second source>
, source_b AS (
    SELECT *
    FROM {{ ref('stg_source__table_b') }}
)

-- Overview: <brief description of transformation>
, transformed AS (
    SELECT
        -- IDs (uid, surrogate keys, foreign keys)
        ---------------------------------
        -- Dates
        ---------------------------------
        -- Dimensions
        ---------------------------------
        -- Metrics
    FROM source_a
    LEFT JOIN source_b
        ON source_a.id = source_b.id
)

, final AS (
    SELECT *
    FROM transformed
)

SELECT *
FROM final
```

## Steps

1. Ask the user what sources/staging models this mart will consume
2. Read the referenced staging models to understand available columns
3. Create the mart model applying:
   - All refs at the top in simple CTEs with `SELECT *`
   - Descriptive CTE names
   - Leading commas
   - Explicit `AS` for all aliases
   - Column aliases prefixed with table alias in JOINs
   - `GROUP BY ALL` for aggregations
   - Column order: IDs, dates, dimensions, metrics
   - A `final` CTE before the closing `SELECT * FROM final`
4. Create/update the models YAML with column descriptions

## Rules

- Only use `{{ ref() }}` in mart; use `{{ source() }}` only when the source comes from another dbt project
- Tag must correspond to the source with lowest update frequency
- Business rules and JOINs belong here
- Prefer `table` or `incremental` materialization
- No `ORDER BY`, no `DISTINCT`
- Use `dbt_utils.generate_surrogate_key()` for composite keys
- Read the full style guide at: `.claude/skills/dbt-style-guide/SKILL.md`
